px64 — Lightweight DOM Binding & Scope Engine

Bind plain JavaScript objects to your HTML using data-bind. Tiny, dependency-free reactivity with optional two-way input, list/table rendering, and delegated events. Ship real UIs without a framework.

View on GitHub See Live Demo

MIT licensed • Zero deps • Works in any modern browser

Elevator Pitch

  • Simple: no build, no VDOM—just HTML + JS.
  • Reactive: DOM updates when your data changes.
  • Pragmatic: two-way inputs (value:), delegated events (tap:), lists & tables.
  • Composable: add custom binders for your apps.

Why px64?

1

Zero Dependencies

Drop in a single script. Works as a global, CommonJS, or AMD module.

2

Data-Bind HTML

Use data-bind attributes to wire text, html, attrs, classes, inputs, lists & tables.

3

Shallow Observables

Plain objects become reactive via px64.observable. Keep your mental model simple.

4

Two-Way Where It Matters

value: binder keeps inputs synced; everything else stays predictably one-way.

5

Lists & Tables

list: and table: binders with paging/sorting via px64.listState.

6

Custom Binders

Extend with px64.addBinder(name, fn) to fit your project perfectly.

Hello World — Live & Code, Side-by-Side

Live Preview

Your name

Try typing and clicking “Greet”. This uses value: (two-way input) and tap: (delegated click) plus text: bindings.

Code (open example) 
<div id="hello-app">
  <h3 data-bind="text:title"></h3>

  <input data-bind="value:name" placeholder="Your name">
  <button data-bind="tap:greet">Greet</button>

  <p data-bind="text:greeting"></p>
</div>

<script src="px64.js"></script>
<script>
  const state = {
    title: "px64 • Hello",
    name: "",
    greeting: "",
    greet() {
      const who = (this.name || "").trim() || "world";
      this.$set("greeting", `Hello, ${who}!`);
    }
  };
  px64.bind("#hello-app", state);
</script>
